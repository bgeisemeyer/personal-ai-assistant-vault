# Safety and Liability

## Position
This framework should encourage conservative defaults while clearly stating that it does not guarantee safe or correct behavior.

## Safety Principles
- Ask before risky external actions.
- Treat sensitive domains conservatively.
- Keep public framework files separate from private runtime data.
- Store explicit boundaries in files, not only in sessions.

## Liability Position
- No warranty
- No security guarantee
- No responsibility for user decisions or AI mistakes
- No responsibility for damages caused by integrations, automations, or incorrect outputs

## V1 Scope
- Safety model documented in files
- No integrated external action layer
